Callbacks

According to Wikipedia:
"In computer programming, a callback is a piece of executable code that is passed as an argument to other code, which is expected to call back (execute) the argument at some convenient time."

In other words, in Java, we can pass our functions (presented by objects) in a method/function as its arguments.

Let's take an example. Since Java 8, any collection has the forEach() method. The method accepts Consumer and calls it for each element in the collection.

// our collection with fruits
List<String> fruits = Arrays.asList("apple", "pear", "orange", "banana");

// it will print all fruits in the collection
fruits.forEach(System.out::println);
Of course, we can pass a more difficult logic as a callback. 

fruits.forEach(fruit -> {
    String doubleFruit = fruit.concat(fruit).toUpperCase();
    if (doubleFruit.length() > 10) {
        System.out.println(doubleFruit);
    }
});

The result is shown below. 
ORANGEORANGE
BANANABANANA

Note. In functional programming, a function (including methods in Java) that accepts or returns another function is called a higher-order function. A lot of features such as function composition, currying, monads and some others based on this idea.
